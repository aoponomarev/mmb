# Исследование: реальные лимиты CoinGecko Public API

**Дата:** 2026-02-27  
**Метод:** консольное тестирование через Node.js HTTPS-запросы  
**Эндпоинт:** `GET /api/v3/coins/markets?vs_currency=usd&ids=...&price_change_percentage=1h,24h,7d`  
**Размер чанка:** 25 монет на запрос

---

## Вывод (коротко)

> **Публичный API CoinGecko без ключа: ~3 запроса за 60 секунд (скользящее окно).**  
> Безопасная задержка между запросами — **21 секунда**.  
> При превышении лимита — HTTP 429 с заголовком `Retry-After: 60`.

## Механизм лимитирования CoinGecko

- **Тип:** скользящее окно (sliding window), ~60 секунд
- **Лимит:** 3 запроса в окне
- **Ответ при превышении:** HTTP 429, заголовок `Retry-After: 60`
- **Блокировка:** по IP-адресу (не по User-Agent)
- **Примечание:** смена User-Agent не помогает — CoinGecko банит по IP

### Почему первый запрос без User-Agent возвращал 403?

Без заголовка `User-Agent` (или с дефолтным `node-fetch`) CoinGecko возвращает **403 Forbidden** — это защита от скриптов. Браузерный User-Agent (`Mozilla/5.0 ...`) обходит эту защиту.

## Применённые изменения в коде

### `core/config/data-providers-config.js`
```js
rateLimit: {
    // Реальный лимит публичного API CoinGecko: ~3 req/60s (скользящее окно).
    // Протестировано опытным путём: безопасная задержка — 21s между запросами.
    requestsPerMinute: 2,
    requestsPerSecond: 0.048
}
```

### `core/api/data-providers/coingecko-provider.js`
```js
// Дефолтная задержка между чанками:
const chunkDelayMs = Number.isFinite(options.chunkDelayMs)
    ? Math.max(0, Math.floor(options.chunkDelayMs))
    : (isFile ? 21000 : 5000);
```

### `app/components/coin-set-load-modal-body.js`
```js
chunkSize: 25,
chunkDelayMs: 21000,
```

## Практические последствия для UX

| Монет | Чанков (по 25) | Время загрузки |
|---|---|---|
| 25 | 1 | ~0.5с |
| 50 | 2 | ~21.5с |
| 100 | 4 | ~63с |
| 250 | 10 | ~189с (~3 мин) |

- Монеты появляются в таблице **по мере загрузки каждого чанка** (стриминг)
- Кнопка **«Остановить»** позволяет прервать загрузку в любой момент
- **2-часовой кэш** (`_cachedAt`) позволяет не перезапрашивать свежие монеты

## Рекомендация для ускорения

Единственный реальный способ ускорить загрузку — **CoinGecko Demo или Pro API ключ**:

| Тариф | Лимит | Задержка | 250 монет |
|---|---|---|---|
| Public (без ключа) | ~3 req/min | 21s | ~3 мин |
| Demo (бесплатный ключ) | 30 req/min | 2s | ~20с |
| Pro (платный) | 500 req/min | 0.12s | ~1.5с |

Поле для ввода ключа уже есть в UI (`ai-api-settings.js`). Нужно только подключить его к `CoinGeckoProvider`.
