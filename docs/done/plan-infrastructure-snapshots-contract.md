# План: Контракт инфраструктурных снимков (rollback-safe deployments)

## Цель

Обязать создание и хранение **стабильных снимков** для каждого деплоируемого комплекса с возможностью отката. **Единообразие:** один формат и одна корневая структура для всех источников (Cloudflare, Yandex, и т.д.). Обязательные провязки: скиллы (какие задействованы), казуальности (причины изменений), цели функциональности деплоя. Снимки создаются **только для успешных** конфигураций; неудачные попытки не архивируются. Макро-спека: id:ais-8b2f1c.

## Scope (что покрывается)

| Компонент | Расположение кода | Что сохранять в снимке |
|-----------|-------------------|------------------------|
| Cloudflare Edge API (Worker) | is/cloudflare/edge-api/ | src/ (копия), wrangler.toml, migrations/ при изменении; env (имена), bindings (имена и типы); **значения настроек** (memory, timeout и т.д.) |
| Yandex: market-fetcher | is/yandex/functions/market-fetcher/ | index.js, package.json (и lock); env (имена); **значения настроек** (memory, timeout, триггеры cron); migrations/ при изменении схемы БД |
| Yandex: api-gateway (coins-db-gateway) | is/yandex/functions/api-gateway/ | index.js, package.json, spec.yaml (если есть); env (имена); **значения настроек** (memory, timeout, allow-unauthenticated-invoke и т.д.) |
| Схема БД (PostgreSQL mbb_db) | Вне репо | Описание/миграции на момент стабильной версии — в README снимка соответствующего Yandex-артефакта или отдельный schema-файл при изменении |

Дополнительно при появлении: любые другие Workers, Yandex Functions, N8N, внешние датасеты и сервисы — тот же контракт: код/конфиги, env (имена), **значения несекретных настроек** (всё, что задаётся в консоли любого сервиса). Секреты — никогда.

## Принцип отсева

- **Сохраняем только после успешного деплоя и проверки.** Перед созданием снимка обязательны: успешный deploy + минимальная верификация (health-check, invoke или один запрос к API).
- Неудачные попытки (deploy failed, проверка не прошла) **не** кладутся в deployments/ в виде стабильных снимков. Не засоряем архив промежуточными билдами.
- Снимок создаётся **автоматически** в явном шаге конвейера деплоя сразу после успешного деплоя и проверки; если деплой или проверка падают после создания рабочего снимка, конвейер обязан автоматически удалить этот снимок и зафиксировать причину дефекта (ссылка на задачу/коммит + краткое описание, см. id:doc-f1a4d3).

6. **Проверка** — `npm run skills:check`, `npm run preflight`.

## Риски и ограничения

- **Секреты:** В снимки попадают только имена env и контракт (обязательный/опциональный). **Значения секретов** (пароли, API-ключи) — никогда. **Значения несекретных настроек** (memory, timeout, триггеры и т.д.) — обязательны к хранению.
- **Размер репо:** Копии src/ и package.json добавляют объём; при длинной истории можно ввести политику хранения только последних N снимков или перенос старых в архив вне репо (опционально, не в первой итерации).
